# 思源别名处理器 (SiYuan Alias Processor)

[English Version](./README_en_US.md)

这是一款为思源笔记设计的专业批量别名同步工具。它采用底层物理路径遍历技术，能够精准、安全地识别并同步文件夹及其子文档的别名属性。

## 核心工程逻辑

### 1. 递归深度同步 (Recursive Deep Sync)
不同于传统的浅层扫描，本插件采用**递归深度遍历**技术：
- **全层级识别**：利用思源文件树 API，自动钻取任意深度的子文件夹。
- **物理路径锚定**：基于文档的内部物理路径 (`path`) 而非易变的 `hpath` 进行定位，确保 100% 的识别精度。

### 2. 只读式安全解析 (Read-Only Parsing)
为了极致保护用户数据，插件遵循**内容不落地**原则：
- **绝不修改正文**：脚本仅以“只读”方式扫描文档块，**永久禁用**了对文档内容的 `updateBlock` 操作。
- **无损属性合并**：新识别的别名会与已有别名进行智能合并，确保手动设置的别名不会被覆盖或丢失。

### 3. 穿透索引限制 (Index Bypass)
针对大型笔记本可能出现的数据库索引不全（如大量日记未收录）的问题：
- **文件树回退机制**：当 SQL 索引缺失时，插件会自动通过文件系统 API 补全缺失文档，确保识别数量与磁盘文件数一致。

### 4. 别名来源支持
- **YAML 属性**: 支持 `alias: [a, b]` 或多行列表格式。
- **行内属性**: 支持 `alias:: 值` 格式。
- **智能清理**: 自动剥离 YAML 列表中的破折号 (`-`) 等格式符号。

## 如何使用

1. 点击顶栏的 **同步图标** 或按下快捷键。
2. 插件会自动定位当前所在文件夹，并统计包含当前文档在内的所有子文档数量。
3. 在弹出的**安全确认窗**中核对文档数量。
4. 点击确定，插件将后台静默完成所有别名的合并与同步。

## 安全特性
- **根目录保护**：禁止在笔记本根目录下运行，防止大范围误操作。
- **进度可视化**：实时显示当前处理的文档路径及总体进度。

## 语言支持
支持 **中文** 和 **英文**。
